# Deployment Guide - CaseFlow Full-Stack Judiciary Management System

Follow this step-by-step guide to deploy the backend and frontend services, connecting them to a cloud-hosted PostgreSQL database.

---

## 1. Database Provisioning (Neon or Supabase)

### Option A: Neon (Easiest for quick PostgreSQL setups)
1. Go to [Neon.tech](https://neon.tech/) and sign up.
2. Create a new project named `caseflow`.
3. Select PostgreSQL version (default is fine) and choose a region close to your target audience.
4. Copy the connection string provided. It should look like:
   `postgresql://[user]:[password]@[host]/[database]?sslmode=require`

### Option B: Supabase
1. Go to [Supabase.com](https://supabase.com/) and create a new project.
2. Once the project is active, go to **Project Settings** -> **Database**.
3. Under the Connection string section, select **URI** and copy the string.

---

## 2. Backend Deployment (Render.com)

1. Go to [Render](https://render.com/) and log in.
2. Click **New** -> **Web Service**.
3. Connect your GitHub repository.
4. Configure the following service settings:
   - **Name**: `caseflow-backend`
   - **Language**: `Node`
   - **Root Directory**: `server`
   - **Build Command**: `npm install && npx prisma generate`
   - **Start Command**: `node src/server.js`
5. Click **Advanced** and add the following **Environment Variables**:
   - `DATABASE_URL`: *Your Neon/Supabase Connection String*
   - `JWT_SECRET`: *Choose a strong secret phrase*
   - `PORT`: `10000` (Render binds automatically, but setting this is safe)
6. Change the database provider in `server/prisma/schema.prisma` if deploying (replace `sqlite` with `postgresql`):
   ```prisma
   datasource db {
     provider = "postgresql"
     url      = env("DATABASE_URL")
   }
   ```
7. Run the initial database setup. You can run Prisma pushes by logging in to the Render shell or from your local machine pointed to the database:
   ```bash
   npx prisma db push
   ```

---

## 3. Frontend Deployment (Vercel)

Vercel is the recommended choice for hosting Vite applications.

1. Go to [Vercel](https://vercel.com/) and connect your GitHub account.
2. Select your repository.
3. Configure the following project settings:
   - **Framework Preset**: `Vite`
   - **Root Directory**: `./` (leave default)
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
4. Set up a Vercel URL routing fallback to prevent React Router page load errors (404s on page refresh). Add a `vercel.json` file in the root directory:
   ```json
   {
     "rewrites": [
       { "source": "/api/(.*)", "destination": "https://your-render-backend-url.onrender.com/api/$1" },
       { "source": "/(.*)", "destination": "/index.html" }
     ]
   }
   ```
5. Note: Replace `https://your-render-backend-url.onrender.com` in `vercel.json` with the actual backend URL generated by Render.

---

## 4. Local Run Instructions (Development Mode)

### Step 1: Run the Backend
```bash
cd server
npm install
npx prisma db push
npm run dev
```

### Step 2: Run the Frontend (in a separate terminal)
```bash
npm install
npm run dev
```
Open [http://localhost:8080](http://localhost:8080) to test the app.
